# 背景

随着数字经济和区块链技术的发展，个人与平台之间的互动缺乏切实的记录方式和价值赋予。无形资产和行为难以量化，而去中心化、透明化、高效化和用户自主权正成为现代生态系统的核心需求。在当前Web3项目中，用户参与度低、激励机制碎片化、治理无效以及<DynamicLangLink className='text-primaryHue' isExternal showAnchorIcon basePath='/bvi' subPath='/identity'>身份</DynamicLangLink>自由度有限等挑战，往往阻碍了生态系统的长期健康发展。

为了解决这些问题，Xone的主要目标和愿景是构建一个理想的BVI（行为价值激励）系统。该系统旨在将每一个用户行为转化为可衡量的激励分数，并为用户提供一个自由、开放且独特的<DynamicLangLink className='text-primaryHue' isExternal showAnchorIcon basePath='/bvi' subPath='/identity'>身份</DynamicLangLink>平台——最终实现 **"将无形行为转化为有形价值"** 的目标。以行为为锚点，以<DynamicLangLink className='text-primaryHue' isExternal showAnchorIcon basePath='/bvi' subPath="/identity">身份</DynamicLangLink>为纽带，以治理为基础，BVI系统致力于推动一个公平、透明和高效的生态系统。

## 理念与目标

### 2.1 行为估值与价值积累

BVI的核心思想借鉴了<Link className='text-primaryHue' isExternal showAnchorIcon href="https://zh.wikipedia.org/wiki/%E8%8A%9D%E9%BA%BB%E4%BF%A1%E7%94%A8">"信用评分"（如芝麻信用）</Link>的概念，但超越了信誉评估的范畴。它是一个全面的激励系统，旨在量化用户在链上的行为和贡献。每一个行动——无论是交易、跨链交互还是社区参与——都可以转化为相应的激励分数（GV）。这种方法为原本无形的行为赋予了明确的价值，这些分数作为奖励累积在用户的账户中。

### 2.2 激励与治理

BVI系统引入了两种主要的激励类型，确保参与者的经济回报和去中心化治理，以避免权力集中：

- **经济激励**：累积的GV可以兑换原生链代币<DynamicLangLink className='text-primaryHue' showAnchorIcon basePath='/study' subPath='/xoc'>XOC</DynamicLangLink>，为用户提供切实的经济回报。
- **治理激励**：GV也可以转化为投票权（GVOTING），使用户能够参与平台重大决策，并助力生态系统优化和资源分配。

在BVI中，<DynamicLangLink className='text-primaryHue' isExternal showAnchorIcon basePath='/bvi' subPath="/identity">身份</DynamicLangLink>不再局限于固定标签，而是一个灵活、开放且可扩展的<DynamicLangLink className='text-primaryHue' isExternal showAnchorIcon basePath='/bvi' subPath='identity'>身份</DynamicLangLink>系统。每个用户的区块链地址不仅是一个账户，更是整个生态系统中<DynamicLangLink className='text-primaryHue' isExternal showAnchorIcon basePath='/bvi' subPath="/identity">身份</DynamicLangLink>、声誉、历史贡献以及DAO或DApp隶属关系的凭证。通过整合去中心化身份（<Link className='text-primaryHue' isExternal showAnchorIcon href="https://ethereum.org/en/decentralized-identity/">DID</Link>）技术，BVI提升了安全性、互操作性和用户自主权——从而释放<DynamicLangLink className='text-primaryHue' isExternal showAnchorIcon basePath='/bvi' subPath="/identity">身份</DynamicLangLink>的全部价值。

### 4.1 选择身份方案

**每个加入Xone的地址（用户）** **默认被识别为普通用户**。在此默认状态下，任何链上行为都不会产生实际价值。由于Xone是一个EVM兼容的Layer-1链，此类地址与以太坊、BNB智能链、Arbitrum等其他公链上的地址并无区别。

Xone的独特之处在于将其用户划分为三大<DynamicLangLink className='text-primaryHue' isExternal showAnchorIcon basePath='/bvi' subPath="/identity">身份</DynamicLangLink>类别：**个人**、**组织**和**项目**。

在传统行业中，这会被视为"打标签"，从逻辑上讲也确实如此。然而，Xone的意图超越了打标签。它旨在为Xone生态系统内的每一位用户提供一个可视化且有意义的**<DynamicLangLink className='text-primaryHue' isExternal showAnchorIcon basePath='/bvi' subPath="/identity">身份</DynamicLangLink>**，以便根据每个用户的角色和期望进行价值权益分配。

**选择<DynamicLangLink className='text-primaryHue' isExternal showAnchorIcon basePath='/bvi' subPath="/identity">身份</DynamicLangLink>是参与BVI的先决条件**——这是一个强制性步骤。每个地址**只能选择一个<DynamicLangLink className='text-primaryHue' isExternal showAnchorIcon basePath='/bvi' subPath="/identity">身份</DynamicLangLink>**，并且一旦选定，**无法更改**。

### 4.2 参与特定行为

为了规范和可视化每个拥有<DynamicLangLink className='text-primaryHue' isExternal showAnchorIcon basePath='/bvi' subPath="/identity">身份</DynamicLangLink>的地址（BVI参与者）的贡献，Xone将这些行动定义为**行为**，或者更实际地说，**任务**。

行为形式多样。有些行为需要**在指定项目内进行特定交易或满足特定标准**，而其他行为则是**由生态系统内的DApp开发者或运营者分配的任务**。为了支持开发者，**Xone将提供开放的API和SDK**，以创建个性化和灵活的集成BVI的DApp。

目前，Xone仍处于早期和脆弱阶段。与成熟的公链相比，Xone在工具、生态系统成熟度和社区规模方面仍然落后。为了解决这个问题，Xone的运营者引入了**BVI <DynamicLangLink className='text-primaryHue' isExternal showAnchorIcon basePath='/bvi' subPath="/season">赛季</DynamicLangLink>** 的概念，每个**赛季都包含有资格获得BVI奖励的指定行为**。

每个赛季根据生态系统目标为**各种行为分配不同的权重**。例如，在<DynamicLangLink className='text-primaryHue' isExternal showAnchorIcon basePath='/bvi' subPath="/season">第1赛季</DynamicLangLink>中，A、B、C三种行为可能贡献GV，权重分别为60%、25%和15%，这表明行为A是本赛季的重点。**未来的赛季（例如<DynamicLangLink className='text-primaryHue' isExternal showAnchorIcon basePath='/bvi' subPath="/season">第2赛季</DynamicLangLink>）** 将根据市场、运营和生态系统的反馈调整行为权重——从而定义了动态的**BVI <DynamicLangLink className='text-primaryHue' isExternal showAnchorIcon basePath='/bvi' subPath="/season">赛季</DynamicLangLink>框架**。

### 4.3 基于纪元的结算

当**BVI参与者完成赛季特定任务**后，结算将在**每个新<DynamicLangLink className='text-primaryHue' isExternal showAnchorIcon basePath='/study' subPath='/epoch'> 纪元 </DynamicLangLink>开始时**进行，基于**上一个<Link className='text-primaryHue' isExternal showAnchorIcon basePath='/study' subPath='/epoch'> 纪元 </Link>** 内完成的行为。

GV奖励根据**赛季行为权重**和**个体用户贡献**进行分配。为了维持**每个<DynamicLangLink className='text-primaryHue' isExternal showAnchorIcon basePath='/study' subPath='/epoch'> 纪元 </DynamicLangLink>固定的GV奖励池总量**，Xone采用了**截断机制**：如果用户的行为贡献占比**≤ 0.001%**，则**不会获得奖励**。

此外，在**"零赛季"**（没有活跃BVI赛季的时期），所有**原计划用于该<DynamicLangLink className='text-primaryHue' isExternal showAnchorIcon basePath='/study' subPath='/epoch'> 纪元 </DynamicLangLink>奖励池的XOC代币将被发送至销毁地址**——永久销毁。该机制旨在调节XOC的流通供应量并支持价格稳定。

需要注意的是，**GV具有有效期**。在一个<DynamicLangLink className='text-primaryHue' isExternal showAnchorIcon basePath='/study' subPath='/epoch'> 纪元 </DynamicLangLink>内获得的GV必须在**20个<DynamicLangLink className='text-primaryHue' isExternal showAnchorIcon basePath='/study' subPath='/epoch'> 纪元 </DynamicLangLink>内**被领取或使用。例如，在<DynamicLangLink className='text-primaryHue' isExternal showAnchorIcon basePath='/study' subPath='/epoch'> 纪元 </DynamicLangLink> #1中获得的GV，如果未被使用，将在**<DynamicLangLink className='text-primaryHue' isExternal showAnchorIcon basePath='/study' subPath='/epoch'> 纪元 </DynamicLangLink> #21开始时被销毁**。

### 4.4 获得有形价值

既然我们已经概述了BVI的核心逻辑，一个主要问题仍然存在：**用户参与BVI究竟能获得什么有形价值？** 为什么不直接奖励XOC或其他代币？

这是Xone的运营者和开发者仔细考虑过的。目标是将Xone建设成一个**自由、开放、公平、透明和高效的生态系统**。从一开始就将行为价值直接与代币挂钩会**与BVI的设计理念相悖**。

GV**不是货币，而是一个占位符**——一种代表潜在价值的中间件。它可能以**积分、能量、优惠券、凭证、忠诚度代币、股份、期权**的形式出现，或者任何您熟悉的类似机制。当用户通过BVI赚取GV时，**他们获得的是一个可以解锁未来有形价值的凭证**。

此外，生态系统内的开发者可以将GV作为其**自定义项目策略**的一个组成部分，使其适应他们自己的运营或奖励框架。

## 如何计算

在以上内容中，我们了解到每次参与行为都会根据特定权重以及自身参与行为的占比进行结算。那么我们如何才能知道每次参与行为可以获得多少GV呢？

<div className="overflow-x-auto">
```math
GV_{i,b}(E) = \left( \frac{x_{i,b}(E)}{\sum_{u} x_{u,b}(E)} \right) \times w_b \times \text{TotalGV}(E)
```
</div>

<Callout>
**含义:**

- **$GV_{i,b}(E)$:** 用户 $i$ 在<DynamicLangLink className='text-primaryHue' isExternal showAnchorIcon basePath='/study' subPath='/epoch'> 纪元 </DynamicLangLink> $E$ 中因行为 $b$ 获得的奖励数量。
- **${x_{i,b}(E)}$:** 用户 $i$ 在<DynamicLangLink className='text-primaryHue' isExternal showAnchorIcon basePath='/study' subPath='/epoch'> 纪元 </DynamicLangLink>  $E$ 中对行为 $b$ 的贡献值（例如，完成任务次数、贡献数据量等）。
- **${\sum_{u} x_{u,b}(E)}$:** 所有用户（用下标 $u$ 表示）对行为 $b$ 的总贡献。
- **$w_b$:** 行为 $b$ 的权重，用于调整不同行为的奖励比例（例如：重要的行为权重更高）。
- **${TotalGV}(E)$:** 当前<DynamicLangLink className='text-primaryHue' isExternal showAnchorIcon basePath='/study' subPath='/epoch'> 纪元 </DynamicLangLink> $E$ 中可以分发给所有用户的总奖励数量。

</Callout>

**逻辑**:

1. **计算用户贡献比例:**

<div className="overflow-x-auto">
```math
\frac{x_{i,b}(E)}{\sum_{u} x_{u,b}(E)}
```
</div>

用户 $i$ 的贡献值占行为 $b$ 中所有用户总贡献值的比例。（例如：如果用户贡献了 100 点，总贡献为 1000 点，则比例为 10.00%）。

2. **按权重分配奖励:**

将该比例乘以权重 $w_b$，以反映不同行为的价值差异。（例如：如果 $w_b = 0.5$，则用户对此行为的奖励比例降至 5.00%）。

3. **总奖励分配:**

最后，乘以 ${TotalGV}(E)$，将比例转换为实际的奖励数量。（例如：如果总奖励是 1000 GV，用户将获得 $5\% \times 1000 = 50$ GV）。

### 1. 多行为求和公式

<div className="overflow-x-auto">
```math
GV_i(E) = \sum_{b=1}^{B} GV_{i,b}(E) = \sum_{b=1}^{B} \left[ \left( \frac{x_{i,b}(E)}{\sum_{u} x_{u,b}(E)} \right) \times w_b \times \text{TotalGV}(E) \right]
```
</div>

<Callout>
**含义:**

- **$GV_i(E)$:** 用户 $i$ 在<Link className='text-primaryHue' isExternal showAnchorIcon href="/study/epoch"> 纪元 </Link> $E$ 中因所有行为获得的总奖励。
- **$B$:** 当前纪元包含的行为类型总数（例如，行为1是发帖，行为2是评论等）。
- 其他变量定义与单一行为公式中相同。

</Callout>

**逻辑**:

1. **计算每个行为的奖励:**

对于每个行为 $b$（从 1 到 $B$），根据单一行为公式计算用户 $i$ 的奖励 $GV_{i,b}(E)$。

2. **累加所有行为的奖励:**

将用户在所有行为中的奖励相加，得到总奖励 $GV_i(E)$。（例如：用户行为1获得 50.00 GV，行为2获得 30.00 GV，则总奖励为 80.00 GV）。

### 2. 身份与 N 个动态因子

<div className="overflow-x-auto">
```math
GV_i(E) = \sum_{b=1}^{B} GV_{i,b}(E) = \sum_{b=1}^{B} \left[ \left( \frac{x_{i,b}(E) \cdot V_{i,b} \cdot \text{N}_{i,b}}{\sum_{u}(x_{u,b}(E)\cdot V_{i,b} \cdot \text{N}_{i,b})} \right) \times w_b \times \text{TotalGV}(E) \right]
```
</div>

对于未来，Xone 希望通过不同的<Link className='text-primaryHue' isExternal showAnchorIcon href="/bvi/identity">身份</Link>系统或相应的可扩展的其他生态系统、社区等，来有效享有差异化的权益，因此在 GV 的计算中引入了<Link className='text-primaryHue' isExternal showAnchorIcon href="/bvi/identity">身份</Link>权益 $V$ 和其他权益因子 $N$。${N}_{i,b}$ 动态因子的计算公式如下：

<div className="overflow-x-auto">
```math
N_{i,b} = 1 + \sum_{k=1}^{K} \gamma_{k,b} \cdot R_{k,i,b}
```
</div>

- **$R_{k,i,b}$:** 用户 $i$ 在行为 $b$ 中的第 $k$ 个权益值（例如：NFT 权益、社区贡献、活跃度等）。
- **$\gamma_{k,b}$:** 权益 $k$ 在行为 $b$ 中的权重系数，用于控制不同权益对 $N$ 的影响，未来可通过社区投票动态调整。

目前，这一期望是 Xone 运营方对未来发展的规划和期望；因此，参与 BVI 的用户可以忽略此部分。在实际结算过程中，Xone 将此参数定义为 1，以保持引入前的状态。

## 愿景与规划

我们相信，构建一个以行为为核心、以<Link className='text-primaryHue' isExternal showAnchorIcon href="/bvi/identity">身份</Link>为纽带、以治理为驱动力的 BVI 系统，将为 Web3 生态系统注入新的活力，解决现有痛点，并形成一个用户参与度高、激励透明、治理公平的未来数字经济模式。让每一次链上互动、每一份贡献都获得清晰的激励，无论是交易、跨链还是社区互动，都能转化为用户价值，并借助自由的<Link className='text-primaryHue' isExternal showAnchorIcon href="/bvi/identity">身份</Link>系统和 GVOTING，将治理权力广泛下放给每个参与者，避免单一机构或大户垄断，实现真正的共治共享。
