# Xone Developer Resources

This guide primarily introduces the basics or manuals for developers building on Xone. Written by Xone builders and for builders to use, it is tailored for those ready to start developing. While you may not fully understand <Link className='text-primaryHue' isExternal showAnchorIcon href="/study/xone">what Xone is</Link>, as a developer, your focus shouldn't be on learning what Xone is, but rather on how to build on it. Xone is equivalent to the <Link className='text-primaryHue' isExternal showAnchorIcon href="https://ethereum.org/en/developers/docs/evm/">EVM series</Link>, so the gap between developing on Xone and Ethereum is minimal. However, keep in mind that there are still some differences between Xone and Ethereum.

## Contract Verification

In Xone, we provide <Link className='text-primaryHue' isExternal showAnchorIcon href="https://testnet.xscscan.com/contract-verification">Verify & Publish Contract</Link>. In just three steps, you can verify and publish the contract you built. In Xone Mainnet, you need to delete the <Code color="danger">testnet</Code> in the access URL, for example: <Code color="success">xscscan.com/contract-verification</Code>, <Link className='text-primaryHue' isDisabled showAnchorIcon href="https://xscscan.com/contract-verification">you can also access it directly here</Link>.

## Practice
<Steps>
### **Step 1: Choose Your Preferred Development Tool**

We recommend using tools designed for EVM-based development. They are not only faster but often offer additional features, such as <Link className='text-primaryHue' isExternal showAnchorIcon href="https://hardhat.org/tutorial/debugging-with-hardhat-network.html">debugging Solidity messages</Link> or <Link className='text-primaryHue' isExternal showAnchorIcon href="https://trufflesuite.com/ganache/">providing user interfaces</Link>.

### **Step 2: Debug Before Deployment**

Once local development is complete, debug your DApp using development nodes or Xone TestNet. Focus on specific aspects of Xone Mainnet, such as verifying contract logic or testing token transfers.  

Deploy to the production network only after ensuring your application runs smoothly on the test network. This reduces unnecessary costs, as every transaction on Xone Mainnet incurs a fee.  

### **Step 3: Verify Contract Source**

You don't need to upload the source code to the <Link className='text-primaryHue' isExternal showAnchorIcon href="/developers/explorers">block explorer</Link> during testing, as the test network allows you to send queries and transactions directly from the graphical interface. On the production network, verified contracts help users better understand their purpose.  
</Steps>
