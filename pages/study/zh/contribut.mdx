
向上游仓库提交拉取请求（通常目标分支是 main；维护者会在审查期间通知您任何特殊分支）。

**在 PR 描述中：**

1. 清晰说明变更的动机和范围；
2. 附上截图/屏幕录制（用于前端变更）；
3. 描述验证步骤和潜在影响（标记"破坏性变更"）。
4. 为 PR 添加标签：至少选择"Documentation/Website/Assets"中的一个；其他标签（bug、enhancement、good first issue 等）可根据需要添加。

<br/>

如果您尚未准备好审查，请选择"创建草稿拉取请求"；准备就绪后，将其标记为"准备审查"。

<Callout emoji="🤝">
  我们在工作日内分配和审查 PR：

  - 重点关注**准确性**、**质量**和**范围**；

  - 可能会建议更改；

  - 审查后，PR 将被合并并自动部署到相应环境。
</Callout>
