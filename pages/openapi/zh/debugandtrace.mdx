## 调试和追踪

仅付费层级可用。允许追踪交易、检查区块执行情况，以及重放交易或区块以进行详细分析。

[trace_filter](/openapi/debugandtrace/trace_filter) : 根据条件筛选追踪数据。 \
[trace_rawTransaction](/openapi/debugandtrace/trace_rawTransaction) : 重放原始交易以进行调试。 \
[trace_block](/openapi/debugandtrace/trace_block) : 追踪区块的执行过程。 \
[trace_replayBlockTransactions](/openapi/debugandtrace/trace_replayBlockTransactions) : 重放区块交易以进行分析。 \
[trace_replayBlockTransactions#vmTrace](/openapi/debugandtrace/trace_replayBlockTransactionsvmTrace) : 在区块重放期间提供虚拟机级别的追踪。 \
[debug_traceBlockByHash](/openapi/debugandtrace/debug_traceBlockByHash) : 通过哈希追踪区块执行。 \
[debug_traceBlockByNumber](/openapi/debugandtrace/debug_traceBlockByNumber) : 通过区块号追踪区块执行。 \
[trace_transaction](/openapi/debugandtrace/trace_transaction) : 追踪特定交易。 \
[debug_traceTransaction](/openapi/debugandtrace/debug_traceTransaction) : 为交易提供详细的追踪信息。 \
[trace_replayTransaction](/openapi/debugandtrace/trace_replayTransaction) : 重放交易以进行分析。 \
[trace_replayTransaction#vmTrace](/openapi/debugandtrace/trace_replayTransactionvmTrace) : 在交易重放期间提供虚拟机级别的追踪。 \
[trace_callMany](/openapi/debugandtrace/trace_callMany) : 执行多个追踪调用。 \
[trace_get](/openapi/debugandtrace/trace_get) : 检索追踪数据。 \
[trace_call](/openapi/debugandtrace/trace_call) : 执行追踪调用。 \
[debug_traceCall](/openapi/debugandtrace/debug_traceCall) : 追踪特定调用而不改变状态。

这些方法提供了先进的调试工具，用于分析 Xone 上的执行流程和调试问题。
