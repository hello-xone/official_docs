## Debug and trace

Available only on the paid tier. Allows tracing transactions, inspecting block execution, and replaying transactions or blocks for detailed analysis.

[trace_filter](/openapi/debugandtrace/trace_filter) : Filters trace data based on criteria. \
[trace_rawTransaction](/openapi/debugandtrace/trace_rawTransaction) : Replays a raw transaction for debugging. \
[trace_block](/openapi/debugandtrace/trace_block) : Traces the execution of a block. \
[trace_replayBlockTransactions](/openapi/debugandtrace/trace_replayBlockTransactions) : Replays block transactions for analysis. \
[trace_replayBlockTransactions#vmTrace](/openapi/debugandtrace/trace_replayBlockTransactionsvmTrace) : Provides a VM-level trace during block replay. \
[debug_traceBlockByHash](/openapi/debugandtrace/debug_traceBlockByHash) : Traces block execution by hash. \
[debug_traceBlockByNumber](/openapi/debugandtrace/debug_traceBlockByNumber) : Traces block execution by number. \
[trace_transaction](/openapi/debugandtrace/trace_transaction) : Traces a specific transaction. \
[debug_traceTransaction](/openapi/debugandtrace/debug_traceTransaction) : Provides detailed trace information for a transaction. \
[trace_replayTransaction](/openapi/debugandtrace/trace_replayTransaction) : Replays a transaction for analysis. \
[trace_replayTransaction#vmTrace](/openapi/debugandtrace/trace_replayTransactionvmTrace) : Provides VM-level trace during transaction replay. \
[trace_callMany](/openapi/debugandtrace/trace_callMany) : Executes multiple trace calls. \
[trace_get](/openapi/debugandtrace/trace_get) : Retrieves trace data. \
[trace_call](/openapi/debugandtrace/trace_call) : Executes a trace call. \
[debug_traceCall](/openapi/debugandtrace/debug_traceCall) : Traces a specific call without making state changes.

These methods provide advanced debugging tools for analyzing execution flows and debugging issues on [Xone](https://xone.org/).
