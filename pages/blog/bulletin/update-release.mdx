Dear [Xone](https://xone.org/) community members:

In order to further improve the openness, transparency and verifiability of the token release mechanism, the [Xone](https://xone.org/) mainnet has recently completed the Epoch module upgrade, and the relevant token release logic has been optimized and adjusted as follows:

## üîÑ Technical mechanism update: access to the chain-level Epoch module

Since <Link className='text-primaryHue' showAnchorIcon href="/blog/bulletin/hard-fork">hard fork height `6397023`</Link>, the bottom layer of [Xone](https://xone.org/) has officially enabled the standardized <Link className='text-primaryHue' showAnchorIcon href="/study/epoch">Epochs</Link> module. The original [Xone](https://xone.org/) donation and team token release logic cycle came from the custom Epoch cycle judgment (that is, it was realized by contract calculation according to the fast release height), and now it has been fully switched to the use of the chain-level unified Epoch scheduling system for processing (**The release here is defined according to Day**).

This will achieve:

- ‚è± **Unified cycle logic**: All release rhythms are based on the <Link className='text-primaryHue' showAnchorIcon href="/study/epoch#core-functions-of-epoch">Epoch module on the chain (every 86400 S = 1 day)</Link> for precise calculation;

- üìä **Verifiable process**: Release data can be tracked and verified in real time in the <Link className='text-primaryHue' isExternal showAnchorIcon href="https://xonescan.com/address/0x08b48c7e6175f1E3BDa4300dC6ee4FbC00ebfC6d">block browser</Link>;

- üõ† **Modular management**: Combined with the upgraded Hook architecture, release events are synchronized with the network.
